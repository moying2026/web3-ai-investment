## 详细验证

### 1. 全部启动/全部暂停 ✅

**API**: POST /api/system/toggle-all

**测试步骤**：
1. 发送 `{"running": false}` → 返回 "All modules paused" (count: 10)
2. 等待5秒后检查所有模块 → 全部 running=False，lastRun时间戳冻结
3. 发送 `{"running": true}` → 返回 "All modules started"
4. 检查所有模块 → 全部 running=True，lastRun开始更新

**验证结果**：全部10个模块（polling, discovery, ai, trading, trench, social, snapshot, onchain, issuer, extended）均可正确启停。

### 2. 单模块启停 ✅

**API**: POST /api/system/:moduleId/toggle

**测试步骤**：
1. 全部启动状态下，单独暂停 trading → trading.running=False
2. 等待5秒后检查 → 仅trading停止，其余9个模块正常运行
3. 恢复trading → trading.running=True

**验证结果**：单模块启停互不影响，隔离性正确。

### 3. 代理保存/测试 ✅

**API**: 
- POST /api/system/proxy（保存）
- POST /api/system/proxy/test（测试）

**测试步骤**：
1. 读取当前代理 → enabled=true, address=http://127.0.0.1:7897
2. 保存相同设置 → 返回 success=true
3. 测试连通性 → success=true, latencyMs=347ms
4. 刷新页面后设置保持

**验证结果**：代理设置保存和测试功能正常。

### 4. 交易模式切换 ⚠️ 前端缺陷

**发现**：

- ✅ **后端已支持**：`PUT /api/sim/settings` 接受 `auto_mode` 参数（"manual"/"auto"），SQLite持久化
- ✅ **后端读取正确**：`GET /api/sim/settings` 返回 `auto_mode` 字段
- ❌ **前端未对接**：
  - `autoMode` 状态仅 `useState(false)` 初始化，未从后端加载
  - `loadStatus()` 函数加载了 stop_loss_percent 和 take_profit_percent，但**未加载 auto_mode**
  - Switch 切换仅修改本地 state，**未调用后端 API 保存**
  - 刷新页面后 autoMode 重置为 false（手动模式）

**影响**：交易模式切换功能在前端是"假开关"，不具备实际效力。

**修复建议**：
1. `loadStatus()` 中增加 `setAutoMode(s.auto_mode === 'auto')`
2. Switch onChange 中增加 `simApi.updateSettings({ auto_mode: checked ? 'auto' : 'manual' })`

### 5. 设置项持久化 ✅

**测试步骤**：
1. 设置 auto_mode="auto" + stop_loss=-20 + take_profit=50
2. 重启后端服务：`systemctl --user restart web3-backend.service`
3. 等待3秒后读取设置

**重启后结果**：
```
stop_loss_percent: -20 ✅
take_profit_percent: 50 ✅
auto_mode: auto ✅
proxy.enabled: true ✅
proxy.address: http://127.0.0.1:7897 ✅
全部模块 running: true ✅
```

**验证结果**：所有设置项（止盈止损、代理、模块状态）在服务重启后完全保持。
