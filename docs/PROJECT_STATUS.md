## 五、多 Agent 评分架构

### Agent 分工

| Agent | 职责 | 评分范围 |
|:--|:--|:--|
| Risk Agent | 审计数据、合约风险 | 0-20 |
| Market Agent | 热度、话题、KOL | 0-15 |
| Issuer Agent | 历史、迁移率、频率 | 0-15 |
| Onchain Agent | 持仓分布、SM信号 | 0-25 |
| Decision Agent | 综合加权汇总 | 0-100 |

### 决策阈值

| 评分 | 建议 | 说明 |
|:--|:--|:--|
| ≥70 | BUY | 买入信号 |
| ≥50 | HOLD | 持有/观望 |
| ≥30 | WATCH | 关注 |
| <30 | AVOID | 回避 |

### 阈值配置（config/thresholds.ts）

- 发行方：总数>100 高风险、7天>10 批量发币、迁移率<10% 高风险
- 地址：top10≥80% 集中、bundles≥50% 刷号、dev≥20% 跑路风险
- 决策：BUY≥70、HOLD≥50、WATCH≥30
