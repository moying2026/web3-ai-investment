# 文档规范 — Web3 AI 投资决策系统

> 本规范继承自桌面自动化工作台的 `文档规范.md`，规则完全一致。
> 详细规则请参阅：`/home/winnie/openclaw/workspace/projects/项目_桌面自动化工作台/文档规范.md`

## 一、首要约束（强制）

每次修改任何文档时，必须同时完成以下三件事：

| # | 操作 | 必填内容 |
|:--|:--|:--|
| 1 | 更新版本号 | `x.y` → `x.y+0.1`（或按版本更新规则） |
| 2 | 更新时间 | `更新时间：YYYY-MM-DD` 改为当天日期 |
| 3 | 写入 `agent-log.md` | 追加操作日志 |

**违反首要约束等同于任务未完成。**

---

## 二、文档分层模型

| Level | 文档 | 数量 | 说明 |
|:--|:--|:--:|:--|
| Level 1 | `项目总则.md`、`文档规范.md`（本文档） | 2 | 顶层规范 |
| Level 2 | `项目概要.md` | 1 | 项目定位与概述 |
| Level 3 | `项目索引.md` | 1 | 文档索引 + 进度总览 |
| Level 4 | `docs/` 下所有 `.md` | N | 执行文档 |

---

## 三、文档头部标准格式（强制）

每个 `.md` 文件头部必须包含：

### 3.1 强制提醒（所有文档必有，必须在最顶部）

```markdown
<!-- ⚠️ 首要约束：修改本文档后必须同步 → 1. 更新「版本」 2. 更新「更新时间」 3. 同步更新上下游关联文档 4. 写入 agent-log.md 5. 确认没有其他相关文件需要更新时，同步上传 github -->
```

### 3.2 三行头部（强制）

```markdown
> **前置依赖：** 先读 → XXX.md
> **下游依赖：** 修改本文档时需同步更新 → XXX.md
> **定位：** 一句话描述本文档职责
```

### 3.3 版本信息块（强制）

```markdown
> 版本：x.y
> 创建时间：YYYY-MM-DD
> 更新时间：YYYY-MM-DD
> 负责人：姓名
```

## 四、版本更新规则

| 修改类型 | 版本号变化 | 更新时间 | 操作日志 |
|:--|:--|:--|:--|
| 小幅修正 | `x.y` → `x.y+0.1` | ✅ | ✅ |
| 功能新增 | `x.y` → `x.y+0.1` | ✅ | ✅ |
| 功能变更 | `x.y` → `x.(y+1)` | ✅ | ✅ |
| 重大重构 | `x.1` → `(x+1).0` | ✅ | ✅ |

## 五、文件命名规则

| 文档类型 | 命名规则 | 示例 |
|:--|:--|:--|
| Level 1 文档 | 中文，固定名称 | `项目总则.md`、`文档规范.md` |
| Level 2 文档 | 中文，固定名称 | `项目概要.md` |
| Level 3 文档 | 中文，固定名称 | `项目索引.md` |
| 架构文档 | 中文，放在 `docs/architecture/` | `数据采集架构.md` |
| 任务卡 | 中文，放在 `docs/tasks/` | `Phase1_数据采集.md` |
| 工作流程 | 中文，放在 `docs/workflows/` | `模拟盘交易流程.md` |
| 操作日志 | 固定名 `agent-log.md` | 项目根目录 |

## 六、新增文档流程

```
1. 在对应目录下创建文件
2. 写入标准头部（三行 + 版本信息块）
3. 在 项目索引.md 中登记
4. 在 agent-log.md 中记录
```
