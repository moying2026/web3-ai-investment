# 项目概要 — Web3 AI 投资决策系统

## 一、项目背景

AI 团队需要自负盈亏，通过 Web3 一级市场新币首发实现盈利。切入点是 Binance Web3 市场，低投入、高回报、时间短。核心风险是骗局多，需要 AI 精准识别规避。

---

## 二、技术架构

### 2.1 技术栈

| 层 | 技术 | 端口 |
|:--|:--|:--|
| 后端 | Node.js + Express + SQLite | 3500 |
| 前端 | Vite + React + TypeScript + Ant Design + ECharts | 3500 |
| 数据源 | Binance Web3 API + CoinGecko + RPC | - |
| 交易工具 | Binance Agentic Wallet CLI (baw) | - |

### 2.2 核心模块

| 模块 | 功能 | 状态 |
|:--|:--|:--|
| 数据采集 | 3秒轮询 Binance API，4链覆盖（BSC/Solana/Base/ETH） | ✅ 已完成 |
| 新币发现 | 90秒轮询，新币自动入库 | ✅ 已完成 |
| AI 评估 | 5维评分（安全/聪明钱/社交/发行方/流动性） | ✅ 已完成 |
| 模拟盘 | 全量买入 + 预算控制 + 止盈止损 | ✅ 已完成 |
| 前端页面 | Dashboard/交易/代币详情/规则引擎/发行方画像 | ✅ 已完成 |
| 实盘交易 | 使用 baw CLI 执行链上交易 | ⏳ 待对接 |

### 2.3 数据流

```
Binance API → 数据采集 → tokens 表
    ↓
AI 评估 → ai_analysis 表
    ↓
模拟盘下单 → sim_trades 表
    ↓
止盈止损 → 检查价格 → 平仓
    ↓（未来）
实盘交易 → baw CLI → 链上执行
```

## 四、关键业务逻辑

1. **未迁移代币是重点**：迁移成功涨幅 1000%+
2. **小投入策略**：不破坏项目方计划
3. **7个关键分析维度**：发行方历史 > 广告推广 > 项目地址 > 话题热度 > KOL > 庄家持币 > 捆绑器
4. **AI 优势**：秒级完成发行方历史、合约风险、社交热度综合判断
5. **模拟盘全量买入**：不论 AI 评分，所有新币都买入，后续验证 AI 准确率
